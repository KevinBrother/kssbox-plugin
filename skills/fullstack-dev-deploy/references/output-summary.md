# 输出摘要

## 目标

在 skill 执行完成时，输出一段结构化摘要，让用户快速知道识别结果、假设项、生成文件和验证情况。

## 必须包含的字段

- 识别到的 app 列表
- `all` 映射了哪些 app
- `biz` 映射了哪些 app
- `boss` 映射了哪些 app
- 新建了哪些文件
- 修改了哪些文件
- 哪些地方按假设处理
- 哪些验证已执行

## 推荐输出顺序

1. 项目画像摘要
2. 服务分组摘要
3. 文件变更摘要
4. 假设项摘要
5. 验证摘要

## 项目画像摘要

至少说明：

- 识别到多少个 frontend app
- 识别到多少个 backend app
- 每类 app 的主要技术栈

## 文件变更摘要

至少区分：

- 新建了哪些文件
- 修改了哪些文件
- 哪些旧资产被复用或收敛到新入口

## 假设项摘要

如果存在自动推断，必须显式说明：

- 哪个映射或命令是按假设处理
- 为什么能继续执行
- 用户后续如果要调整，应改哪一层配置

## 验证摘要

必须说明：

- 哪些验证已执行
- 哪些验证通过
- 若有未执行的验证，是因为什么前置条件缺失

## 禁止行为

- 只说“已完成”，不列出具体文件变化
- 不提假设项，导致用户误以为所有判断都已确认
- 不提验证范围，让用户不知道是否真的跑过
