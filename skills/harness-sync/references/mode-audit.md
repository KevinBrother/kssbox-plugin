# 模式：AUDIT Harness 腐烂检测

定期检测 Harness 文件是否过期、是否和实际代码/约定出现漂移，并修复。

## 什么是 Harness 腐烂

随着 AI agent 持续写代码，Harness 会慢慢失效：
- 代码里出现了 AGENTS.md 没提到的新模式，agent 开始复制这些新模式
- 层级边界被侵蚀，dependency-cruiser 规则开始被注释掉
- AGENTS.md 的"禁止事项"已经不禁止了（没人维护，agent 开始绕过）
- 新的团队决定没有写进 ADR，藏在某个 Slack 消息里
- CI 覆盖率阈值被悄悄调低了

Martin Fowler 把这个叫 "Garbage Collection"——不定期清理，Harness 会腐烂。

## 需要用户提供

理想情况：让 skill 能读取仓库文件（通过文件上传或 bash 工具）。

如果只能对话，问：
1. 最近有没有出现 CI 规则被绕过或注释掉的情况？
2. AGENTS.md 上次更新是什么时候？
3. 最近的代码有没有违反原来约定的模式？

## 检测清单

### A. 文件存在性检查

- [ ] `AGENTS.md` 存在于仓库根目录
- [ ] `docs/architecture.md` 存在
- [ ] `docs/conventions.md` 存在
- [ ] `docs/decisions/` 目录存在，且有至少一条 ADR
- [ ] CI 配置存在且包含四类门禁

### B. AGENTS.md 健康度

- [ ] 禁止事项 ≥ 3 条，且是针对这个项目的具体约束（不是"写好代码"这种废话）
- [ ] 禁止事项和 dependency-cruiser 规则一一对应（说禁的，都有机械执行）
- [ ] "已知常见错误"节有内容（空白说明从未维护）
- [ ] "不确定时怎么做"有明确的人工节点说明
- [ ] 项目概述和当前实际技术栈一致

### C. 架构文档健康度

- [ ] architecture.md 的层级图和实际目录结构一致
- [ ] 每层的"不负责"列表有内容（空白说明没有认真写）
- [ ] 外部依赖（第三方集成）已在架构文档中标注

### D. CI 门禁健康度

- [ ] 类型检查未被注释或跳过（`tsc --noEmit` 或 `mypy` 正常运行）
- [ ] linter 的 `--max-warnings=0` 没有被放宽
- [ ] 模块边界检查（dependency-cruiser）未被注释
- [ ] 覆盖率阈值未被悄悄调低（检查 `coverageThreshold` 的数值）
- [ ] 所有门禁失败时都是 `exit 1`（阻断），不是只打 warning

### E. 决策记录健康度

- [ ] 最近 3 个月有重要架构决策但没有对应 ADR → 缺失
- [ ] 已有 ADR 的状态字段是"已确认"而非"草稿"（草稿 = 决策悬空）

### F. 腐烂信号检测

以下任一出现，说明 Harness 开始腐烂：

| 信号 | 含义 | 处理 |
|-----|-----|-----|
| CI 中有 `|| true` 或 `continue-on-error: true` | 门禁被绕过 | 立即修复，找出绕过原因 |
| dependency-cruiser 规则有 `doNotFollow` 异常太多 | 边界被打洞 | 清理例外，或修改架构 |
| AGENTS.md 的禁止事项里有"// TODO: 确认"注释 | 规则不确定 | 决策后更新 |
| coverage 报告里 Service 层覆盖率 < 60% | 测试大量缺失 | 补测试，可能 agent 没遵守要求 |
| ADR 最近更新时间 > 3 个月 但项目活跃 | 决策没有记录 | 补写缺失的 ADR |

## 输出：Harness 健康报告

```
## Harness 健康报告
检测时间：[日期]

### 总体评分：🟡 需要关注（或 ✅ 健康 / 🔴 需要立即修复）

| 维度 | 状态 | 问题数 |
|-----|-----|------|
| A. 文件存在性 | ✅ | 0 |
| B. AGENTS.md | 🟡 | 1 |
| C. 架构文档 | ✅ | 0 |
| D. CI 门禁 | 🔴 | 2 |
| E. 决策记录 | 🟡 | 1 |
| F. 腐烂信号 | 🔴 | 1 |

### 发现的问题

🔴 **立即修复**：
- CI 中 dependency-cruiser 被 `|| true` 绕过（ci.yml 第 34 行）
- coverage 阈值从 80% 被调低到 50%（未记录原因）

🟡 **近期处理**：
- AGENTS.md "已知常见错误"节为空，最近 agent 的错误模式未记录
- 缺少 ADR：上个月引入了 Redis 缓存但没有决策记录

### 修复建议

[按优先级列出具体修复步骤，可直接执行]
```

## 修复执行

发现问题后，询问用户："我发现了 X 个问题，要帮你直接修复吗？"

用户确认后，执行修复并落盘（使用 mode-update 的文件生成规范）。
