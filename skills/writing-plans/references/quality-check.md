# 计划质量检查清单

在输出计划文档前，逐条检查，全部通过才交付。

## 结构完整性

- [ ] 文档有 **目标** 字段，一句话说清楚交付物
- [ ] 文档有 **文件结构表格**，每行都有路径和职责说明
- [ ] 每个 Task 都有 **一句话描述**（`> xxxx`）
- [ ] 每个 Task 有 **关联文件** 字段
- [ ] 文档有 **验收标准** 章节，至少 2 条可执行的检查项

## 任务设计

- [ ] **最小化**：没有把不相关需求塞进一个任务
- [ ] **可测试**：每个 Task（除 Task 0）至少有一个测试步骤
- [ ] **测试先行**：测试步骤在实现步骤之前
- [ ] **依赖标注**：有跨任务依赖的地方已标注 `**依赖：** Task N`
- [ ] **无模糊步骤**：每个步骤都是具体动作，没有"实现功能"这类描述

## 文件结构

- [ ] 没有职责不清的文件名（如 `utils.ts`、`helpers.ts` 未加限定词）
- [ ] 新建文件的路径与现有代码风格一致
- [ ] 修改量较大的文件已标注影响范围

## 落盘检查

- [ ] 文件路径格式：`.plans/YYYY-MM-DD-<feature>.md`（相对于目标项目根目录）
- [ ] 文件已实际写入磁盘（不是只在对话中展示）
- [ ] 已告知用户：**下一步用 executing-plans 指定此文件路径**
