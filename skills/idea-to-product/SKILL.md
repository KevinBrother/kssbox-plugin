# Idea → 完整产品规划 Skill

## 执行流程（严格按顺序）

```
阶段一：对话补全 idea   → 详见 references/phase1-interview.md
阶段二：联网竞品调研    → 详见 references/phase2-research.md
阶段三：输出规划文档    → 详见 references/phase3-chapters.md
                          模板：references/output-template.md
                          推广：references/gtm-playbook.md
阶段四：落盘 + 自检     → 详见 references/save-and-test.md
```

**在每个阶段开始前，先读对应的 reference 文件，再执行。**
**阶段四不可跳过，文档不落盘、不自检，不算完成。**

---

## 阶段一：对话补全 idea

读取 `references/phase1-interview.md` 后开始提问。

核心原则：**先问，不输出，不搜索。** 最多两轮问答，信息足够后主动推进。

---

## 阶段二：联网竞品调研

读取 `references/phase2-research.md`，执行搜索。

核心原则：**必须联网搜索，不能用记忆替代。** 搜索完成后再进入阶段三。

---

## 阶段三：输出完整产品规划文档

读取 `references/phase3-chapters.md`，按文档结构逐章生成内容。
输出格式参考 `references/output-template.md`，推广细节参考 `references/gtm-playbook.md`。

核心原则：**所有章节必须有实质内容，不能只写框架。**

文档结构：
```
一、产品定位
二、用户与痛点
三、竞品分析与市场定位
四、优缺点评估（含优化建议）
五、核心功能与产品流程
六、MVP 范围
七、推广策略
八、收费模式
九、产品增长闭环
十、冷启动行动清单（90天）
十一、PRD 输入摘要
```

---

## 输出质量检查

生成文档前自检，所有项通过才能输出：

- [ ] 竞品数据来自真实联网搜索，不是凭记忆
- [ ] 蓝红海判断有具体依据（融资/竞品数量/用户投诉）
- [ ] 优化建议针对具体劣势，不是"加强差异化"这种废话
- [ ] 推广章节有具体渠道名，不是"社交媒体"
- [ ] 定价有具体数字，对标了至少 2 个竞品价格
- [ ] 增长闭环每个环节有具体机制，含 5 个埋点列表
- [ ] 行动清单每条动词开头、可执行
- [ ] PRD 摘要可以直接作为 PRD skill 的输入
- [ ] 一句话定位 10 秒内让陌生人理解

---

## 特殊情况

| 情况 | 处理方式 |
|-----|---------|
| 用户已有竞品分析 | 补搜用户反馈/定价数据，跳过已知部分 |
| 技术背景用户 | PRD 摘要可加深技术细节 |
| 已有用户/收入 | 重点放增长闭环，冷启动简化 |
| idea 太模糊 | 用 phase1 的引导句式重构后再搜索 |
| 搜索结果匮乏 | 用间接竞品替代，文档注明数据局限性 |
| 用户中途想停止 | 输出已完成的部分，说明后续可继续，保存当前进度 |
| 用户只做部分章节 | 明确跳过哪些章节，聚焦用户指定的部分 |
