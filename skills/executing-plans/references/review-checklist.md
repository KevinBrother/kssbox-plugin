# 步骤一：计划审查清单

在开始执行任何任务前，逐条过以下检查项。

## 必检项

- [ ] 计划文件有明确的 **目标** 字段
- [ ] 文件结构表格存在且每行有路径和职责说明
- [ ] 所有任务都有"一句话描述"
- [ ] 任务间依赖关系已明确标注（或无依赖）
- [ ] 验收标准章节存在，有可执行的检查项

## 可疑信号（发现即停止，向用户提问）

| 信号 | 提问方向 |
|------|--------|
| 某个任务步骤含"实现功能"/"处理逻辑"等模糊描述 | 请补充具体实现要点 |
| 某个文件路径在现有代码库找不到父目录 | 确认路径是否正确，还是需要先创建目录 |
| Task N 依赖 Task M，但 Task M 在 N 之后 | 确认执行顺序 |
| 验收标准里有"手动验证"但没说验证什么 | 请补充具体验证步骤 |

## 审查通过后

建立任务清单，内容结构：

```
Task 0: 初始化         [待执行]
Task 1: 组件名         [待执行]
Task 2: 组件名         [待执行，依赖 Task 1]
...
```

向用户宣告："计划审查通过，开始执行。"
