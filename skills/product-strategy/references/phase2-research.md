# 阶段二：联网竞品调研

**必须执行联网搜索，不能用训练记忆替代。**

## 搜索顺序

### 搜索 1：直接竞品
- 查询词：`[产品核心功能] tool / app / software [当前年份]`
- 目标：找 3-5 个直接竞品，记录名称、定价、核心功能

### 搜索 2：间接替代方案
- 查询词：`how do [目标用户] manage / solve [痛点]`
- 目标：找用户现在实际用的替代方案（含非软件方案，如 Excel、微信群）

### 搜索 3：市场趋势
- 查询词：`[行业] market size / trends [当前年份]`
- 目标：增长/收缩趋势，政策/技术驱动因素

### 搜索 4（可选）：用户真实评价
- 查询词：`[竞品名] review / reddit / alternatives`
- 目标：用户对竞品的真实吐槽 = 差异化机会

## 每个竞品需记录

| 字段 | 说明 |
|-----|-----|
| 名称 + 官网 | |
| 核心功能 | 3 条以内 |
| 定价 | 具体数字，不是"免费增值" |
| 用户规模 | 能找到的数据 |
| 最大优点 | |
| 用户最常抱怨的问题 | 从评测/论坛/App Store 评论找 |

## 蓝红海判断

### 红海信号（竞争激烈）
- 已有 3+ 个融资过千万的玩家
- 搜索词被大量广告占据
- 产品功能高度同质化

### 蓝海缺口信号（机会存在）
- 用户投诉集中在同一个问题
- 某个细分用户群被忽视
- 技术/法规变化带来新需求
- 现有方案太贵/太重，轻量需求无人满足

> ⚠️ 完全蓝海往往意味着没有市场需求。最好的位置是"有竞争但有明显切入缺口"。
