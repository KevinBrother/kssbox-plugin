# 落盘与自检流程

生成文档内容后，必须执行以下两步，不可跳过。

---

## 步骤 A：落盘

将生成的规划文档保存为 Markdown 文件。

**文件命名规则**：`product-plan-[产品名拼音或英文]-[YYYYMMDD].md`
例：`product-plan-indie-crm-20250413.md`

**保存路径**：当前项目根目录下的 `docs/` 目录（即 `./docs/`）。如果用户在其他目录中运行，佄倔询用户是否保存到当前目录。

**执行方式**：用 `create_file` 工具写入，内容为完整的 Markdown 文档。

**写入完成后**：告知用户文件已保存到哪个路径。

---

## 步骤 B：自检

文件落盘后，重新读取刚写入的文件，对照以下清单逐条核查。

### 结构完整性检查

- [ ] 文档包含全部 11 个章节标题（一到十一）
- [ ] 每个章节有实质内容，没有只留占位符（如"[填写]"、"[待补充]"）
- [ ] PRD 输入摘要（第十一章）字段全部填写

### 内容质量检查

- [ ] 竞品对比表有至少 3 个真实竞品，含具体价格数字
- [ ] 蓝红海判断写了具体依据，不是"竞争激烈"这类泛语
- [ ] 推广章节的渠道是具体平台名（如"Reddit r/SaaS"），不是"社交媒体"
- [ ] 文案钩子是真实的句子，不是"[文案示例]"占位
- [ ] 定价表有具体数字，不是"$X/月"占位
- [ ] 增长闭环的信息收集部分列出了 5 个具体埋点行为
- [ ] 行动清单每条以动词开头（如"发布"、"访谈"、"搭建"）
- [ ] 一句话定位符合格式：`[产品名] 是一个帮助 [用户] [解决什么] 的 [类型]`

### 逻辑一致性检查

- [ ] MVP 功能和 PRD 摘要里的功能清单一致，没有矛盾
- [ ] 定价方案和增长闭环里的付费转化路径一致
- [ ] 推广渠道和目标用户画像匹配（如 B2B 产品不应主推小红书）

### 发现问题时

- **轻微问题**（占位符未填、格式不对）：直接在文件中修复，重新保存，不通知用户
- **实质性问题**（章节内容空洞、逻辑矛盾）：修复后告知用户修改了什么
- **无法自动修复**（如搜索结果不足导致竞品数据缺失）：在自检报告中注明，提示用户手动补充

### 输出自检报告

自检完成后，向用户输出简短报告，格式如下：

> **✅ 文档已生成并保存**
> 文件：`product-plan-[名称]-[日期].md`
>
> **自检结果**：通过 X / 共 Y 项
>
> - ✅ 结构完整，11 个章节全部有内容
> - ✅ 竞品数据：找到 X 个真实竞品，含价格
> - ⚠️ [如有问题，列出并说明已修复或需用户处理]
>
> **第十一章"PRD 输入摘要"可作为后续生成开发文档的输入。**
