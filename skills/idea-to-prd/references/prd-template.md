## 5. API 设计

**Base URL**：`/api/v1`  
**认证方式**：Bearer Token（JWT）  
**内容类型**：`application/json`

### 5.1 统一响应格式

```json
// 成功
{ "success": true, "data": { ... } }

// 失败  
{ "success": false, "error": { "code": "ERROR_CODE", "message": "描述" } }
```

### 5.2 认证模块

#### POST /auth/register

**描述**：用户注册  
**认证**：不需要

**请求体**：
```json
{
  "email": "string, 必填, 邮箱格式",
  "password": "string, 必填, 最少8位",
  "name": "string, 必填"
}
```

**响应（201）**：
```json
{
  "success": true,
  "data": {
    "user": { "id": "uuid", "email": "...", "name": "..." },
    "token": "jwt_token"
  }
}
```

**错误**：
- 422：邮箱格式错误 / 密码太短
- 409：邮箱已注册（`EMAIL_ALREADY_EXISTS`）

## 7. 非功能性要求

### 7.1 性能

| 指标 | 目标 | 测量方式 |
|-----|-----|--------|
| 页面首次内容渲染（FCP） | < 2 秒 | Lighthouse |
| API P95 响应时间 | < 300ms | 服务端监控 |
| 并发用户 | 1000 | 压测 |

### 7.2 安全

- 认证：JWT，有效期 7 天，刷新 Token 30 天
- 密码：bcrypt 哈希，最少 8 位
- 传输：全站 HTTPS
- 防护：ORM 防 SQL 注入，框架防 XSS

### 7.3 可用性

- 目标可用性：99.9%（月停机 < 44 分钟）
- 备份：每日自动备份，保留 30 天
